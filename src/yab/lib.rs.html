1//! YAB is **Y**et **A**nother **B**enchmarking framework powered by [`cachegrind`] from the Valgrind tool suite.
2//! It collects reproducible measurements of Rust code (e.g., the number of executed instructions,
3//! number of L1 and L2/L3 cache hits and RAM accesses), making it possible to use in CI etc.
4//!
5//! # Features
6//!
7//! - Supports newer `cachegrind` versions and customizing the `cachegrind` wrapper.
8//! - Supports capturing only instruction counts (i.e., not simulating CPU caches).
9//! - Conditionally injects `CACHEGRIND_{START|STOP}_INSTRUMENTATION` macros (available in `cachegrind`
10//!   3.22.0+) allowing for more precise measurements. See [crate features](#crate-features) below.
11//! - Supports configurable warm-up (defined in terms of executed instructions) before the capture.
12//!
13//! # How to use
14//!
15//! Define a benchmark binary and include it into your crate manifest:
16//!
17//! ```toml
18//! [[bench]]
19//! name = "your_bench"
20//! harness = false
21//! ```
22//!
23//! In the bench source (`benches/your_bench.rs`), define a function with signature `fn(&mut` [`Bencher`]`)`
24//! and wrap it in the [`main!`] macro:
25//!
26//! ```
27//! use yab::Bencher;
28//!
29//! fn benchmarks(bencher: &mut Bencher) {
30//!     // define your benchmarking code here
31//! }
32//!
33//! yab::main!(benchmarks);
34//! ```
35//!
36//! Run benchmarks as usual using `cargo bench` (or `cargo test --bench ...` to test them).
37//!
38//! ## Configuration options
39//!
40//! Run `cargo bench ... -- --help` to get help on the supported configuration options. Some of the
41//! common options are:
42//!
43//! - `--list`: lists benchmarks without running them.
44//! - `--print`: prints results of the latest run instead of running benchmarks.
45//! - `--jobs N` / `-j N`: specifies the number of benchmarks to run in parallel. By default, it's equal
46//!   to the number of logical CPUs in the system.
47//!
48//! # Limitations
49//!
50//! - `cachegrind` has somewhat limited platform support (e.g., doesn't support Windows).
51//! - `cachegrind` uses simplistic / outdated CPU cache simulation to the point that recent versions
52//!   disable this simulation altogether by default.
53//! - `cachegrind` has limited support when simulating multi-threaded environment.
54//! - Even small changes in the benchmarked code can lead to (generally small) divergences in the measured stats.
55//!
56//! # Alternatives and similar tools
57//!
58//! - This crate is heavily inspired by [`iai`](https://crates.io/crates/iai), *the* original `cachegrind`-based
59//!   benchmarking framework for Rust.
60//! - [`iai-callgrind`](https://crates.io/crates/iai-callgrind) is an extended / reworked fork of `iai`.
61//!   Compared to it, `yab` prefers simplicity to versatility.
62//! - Benchmarking APIs are inspired by [`criterion`](https://crates.io/crates/criterion).
63//!
64//! # Crate features
65//!
66//! ## `serde`
67//!
68//! *(Off by default)*
69//!
70//! Derives `serde::{Serialize, Deserialize}` for [`BenchmarkOutput`], [`CachegrindStats`] and related
71//! types. Useful to save benchmark outputs when using a custom [`BenchmarkProcessor`].
72//!
73//! ## `instrumentation`
74//!
75//! *(Off by default)*
76//!
77//! Injects `CACHEGRIND_{START|STOP}_INSTRUMENTATION` macros allowing for more precise measurements.
78//! Requires `cachegrind` 3.22.0+ with dev headers available; see [`crabgrind` docs](https://crates.io/crates/crabgrind)
79//! for details.
80//!
81//! # Examples
82//!
83//! The entrypoint for defining benchmarks is [`Bencher`].
84//!
85//! ```
86//! use yab::{black_box, Bencher, BenchmarkId};
87//!
88//! /// Suppose we want to benchmark this function
89//! fn fibonacci(n: u64) -> u64 {
90//!     match n {
91//!         0 | 1 => 1,
92//!         n => fibonacci(n - 1) + fibonacci(n - 2),
93//!     }
94//! }
95//!
96//! fn benchmarks(bencher: &mut Bencher) {
97//!     // Benchmark simple functions.
98//!     bencher
99//!         .bench("fib_short", || fibonacci(black_box(10)))
100//!         .bench("fib_long", || fibonacci(black_box(30)));
101//!     // It's possible to benchmark parametric functions as well:
102//!     for n in [15, 20, 25] {
103//!         bencher.bench(
104//!             BenchmarkId::new("fib", n),
105//!             || fibonacci(black_box(n)),
106//!         );
107//!     }
108//!     // To account for setup and/or teardown, you may use `bench_with_capture`
109//!     bencher.bench_with_capture("fib_capture", |capture| {
110//!         // This will not be included into captured stats.
111//!         black_box(fibonacci(black_box(30)));
112//!         // This will be the only captured segment.
113//!         let output = capture.measure(|| fibonacci(black_box(10)));
114//!         // This assertion won't be captured either
115//!         assert_eq!(output, 55);
116//!     });
117//! }
118//!
119//! yab::main!(benchmarks);
120//! ```
121//!
122//! [`cachegrind`]: https://valgrind.org/docs/manual/cg-manual.html

149/// Wraps a provided function to create the entrypoint for a benchmark executable. The function
150/// must have `fn(&mut` [`Bencher`]`)` signature.
151///
152/// # Examples
153///
154/// See [crate docs](index.html) for the examples of usage.
155#[macro_export]
156macro_rules! main {
157    ($function:path) => {
158        fn main() {
159            $function(&mut $crate::Bencher::default());
160        }
161    };
162}
